# Constitutional Principle

> A civic institution becomes truly intelligent when every public action can be understood within its complete civic context.

Humanity Union should never present isolated civic objects.

Every initiative, analysis, proposal, decision, implementation, impact, and archive record must remain connected within one transparent constitutional ecosystem.

---

# Purpose

Define the **Capability 02 Integration Layer** — reference-only aggregation that connects existing civic modules without introducing new lifecycles, workflows, or duplicated storage.

The integration layer provides:

- **CivicContextPanel** — contextual awareness across the pipeline
- **RelatedRecord** references — cross-entity navigation
- **CivicPipelineStatus** — informational stage completion
- **Civic breadcrumb** — civic path awareness
- **CivicSearchMetadata** — future global search contract
- **CIVIC_NOTIFICATION_EVENT_REGISTRY** — future notification preparation

---

# Architecture Rules

- Reference-only integration
- No duplicated entity storage
- No duplicated projections
- Reuse existing stores and public APIs
- Module ownership unchanged
- Public Experience Global/Country/Region/Community layouts unchanged

---

# Explicit Exclusions

Do not implement in the integration layer:

- Global search UI
- Notifications UI or delivery
- Recommendation engine
- AI navigation or scoring
- Analytics dashboard
- Personalization, feeds, popularity, or reputation
