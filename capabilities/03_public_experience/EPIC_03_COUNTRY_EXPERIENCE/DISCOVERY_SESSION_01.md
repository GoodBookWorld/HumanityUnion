# 1. Discovery Purpose

Initiate the **architectural discovery** of **Country Experience** within Humanity Union Public Space.

Country Experience represents the **national level** of Humanity Union's public civic space.

It is **not** an independent platform.

It is **not** a separate product, site or navigation universe.

It is a **filtered public representation of one country** within the unified Humanity Union architecture — architecturally named **Geographic Experience at Country scope**.

This discovery session defines the **architectural role** Country Experience must fulfill before specification, content architecture, interaction architecture and implementation planning begin.

Country Experience **extends Global Experience** without introducing a new architectural language.

Epic 02 delivered Global Experience at World scope.

Epic 03 discovery asks how the **same frozen architecture** expresses observable civic reality when the visitor's scope is one country.

This document records discovery intent only.

It does not define implementation.

It does not authorize code, routes, components, visual design or Capability 02 backend changes.

**One architecture.**

**Different geographic scope.**

```
World

↓

Country

↓

Region

↓

Future local levels
```

Architecture remains **identical**.

Only **public datasets**, **scope parameters** and **contextual Identity copy** change.

| What stays the same                                         | What changes                                     |
| ----------------------------------------------------------- | ------------------------------------------------ |
| Header — six primary destinations                           | Scope filter active at Country                   |
| Geographic Navigator                                        | Country selected; Region becomes available entry |

- country-specific header destinations;
- country-specific block catalog;
- country-specific participation pressure patterns.

# 3. Architectural Questions

Discovery must explore and later answer the following questions — without prescribing implementation.

## What makes Country Experience different from Global Experience?

- **Scope parameter** — visitor observes civic activity associated with one country, not the entire world aggregate.
- **Identity framing** — Hero and page context name the country as a participating place within Humanity Union — not as an isolated civic island.
- **Evidence density** — statistics, pipeline distribution, initiatives and map markers reflect national public activity — may include honest sparse or empty states for countries with limited public records.
- **Exploration entry** — map and navigator support descent to Region scope and ascent to World scope — not lateral escape to a different architecture.

Country Experience differs in **dataset and contextual meaning** — not in page anatomy or trust model.

## Which blocks remain identical?

Per frozen Epic 01 and Epic 02 composition, Country Experience reuses the **same block sequence** as Global Experience:

| Order | Architectural block  | Country adaptation                                          |
| ----- | -------------------- | ----------------------------------------------------------- |
| 1     | Hero                 | Country Identity — not platform-only Identity               |
| 2     | Interactive Map      | Country-scoped geographic Evidence — regions within country |
| 3     | Statistics           | National aggregate indicators — World scope label replaced  |
| 4     | Initiative Levels    | National pipeline stage distribution                        |
| 5     | Latest Initiatives   | Recent public initiatives at country scope                  |
| 6     | Registration Gateway | Same Participation stage — calm invitation                  |
| 7     | Footer               | Same Supporting Navigation                                  |

Global chrome unchanged: **Header**, **Geographic Navigator**, **Footer**.

Block **responsibility** is identical.

Block **Evidence** is filtered.

## Which information becomes country-specific?

Discovery candidates — subject to later content architecture:

- country name and scope label in Identity layer;
- country Context Introduction — significance of national civic activity within global Humanity Union;
- national public statistics from filtered projections;
- national participation pipeline distribution;
- latest initiatives associated with the country;
- map Evidence showing regional presence **within** the country;
- optional national media or institutional cross-links where Verified Media and Institutions capabilities supply public-safe country associations.

Country-specific copy must remain **explanatory** — not political, promotional or certifying.

## Which data continues to originate from Capability 02 public projections?

Per `CAPABILITY_02_PROJECTION_INTEGRATION.md`, Country Experience consumes the **same architectural projection contracts** filtered by country scope:

| Block                  | Projection source                                           |
| ---------------------- | ----------------------------------------------------------- |
| Interactive Map        | Public Geographic Participation Projection — country filter |
| Statistics             | Participation Public Statistics Projection — country filter |
| Participation Pipeline | Participation Pipeline Public Projection — country filter   |
| Latest Initiatives     | Initiative Public Projection — country filter               |

Capability 03 **orchestrates and filters**.

Capability 03 **does not** access operational aggregates.

Capability 02 **owns** aggregation rules and public-safe geographic metadata in projection builders.

## How is national identity represented without fragmenting Humanity Union?

Discovery must resolve:

- country named clearly without implying Humanity Union is a federation of separate platforms;
- symbols and flags — if present — as **Identity context**, not authority claims;
- navigation always preserves return path to World scope and global chrome;
- language that presents the country as **one participating place in one connected civic space**.

National Identity strengthens local comprehension.

It must not weaken global coherence.

# 4. Visitor Perspective

## Why would a visitor open a Country Experience page?

Typical intentions:

- understand civic activity **in my country** or **in a country I am researching**;
- move from global overview to **national context** before regional or initiative detail;
- evaluate whether structured participation exists **where I live**;
- follow a map or navigator transition from World to Country deliberately;
- share or bookmark a **country-scoped public URL** as a stable civic reference.

Country Experience serves **local orientation within global architecture** — not national exit from Humanity Union.

## What questions should it answer?

Country Experience should naturally answer:

| Question                                                | Architectural responsibility                                                  |
| ------------------------------------------------------- | ----------------------------------------------------------------------------- |
| **Where am I?**                                         | Country Identity — Hero, scope label, Geographic Navigator                    |
| **Why does this country's civic activity matter here?** | Context Introduction — national significance within Humanity Union            |
| **What is happening in this country?**                  | Evidence — statistics, pipeline, initiatives, map                             |
| **Where can I explore next?**                           | Exploration — regions, related initiatives, header destinations, World return |
| **How may I participate?**                              | Registration Gateway — optional, after observation                            |

These mirror the Five Visitor Questions from `PUBLIC_PAGE_TEMPLATE_STANDARD.md` — unchanged at Country scope.

## How should it relate to Global Experience?

| Relationship         | Discovery intent                                                           |
| -------------------- | -------------------------------------------------------------------------- |
| **Ascent**           | Visitor returns to World Global Experience without losing navigation model |
| **Descent**          | Visitor arrives from World map or navigator — scope change explicit        |
| **Parallel depth**   | Country page is not "more authoritative" than World — it is **narrower**   |
| **Same trust model** | Derived labeling, honest empty states, projection traceability preserved   |

Global Experience orients globally.

Country Experience orients nationally.

Both are Home-class compositions in one Public Space.

## How should it prepare visitors for Region Experience?

Country Experience should:

- present map and navigator entries to **regions within the country** where public data exists;
- explain that Region scope narrows observation further — same architecture, finer filter;
- avoid implying Region is a different product or registration gate;
- preserve reversible navigation — Country ↔ World ↔ Region without penalty.

Country is the **intermediate scope** between World and Region in the frozen geographic hierarchy.

# 5. Geographic Philosophy

Discovery confirms the following geographic philosophy for Country Experience.

**Countries are participating parts of one connected civic space.**

Humanity Union presents national civic activity as **locatable within a global public square** — not as sealed national silos.

| Principle                            | Country Experience expression                                                      |
| ------------------------------------ | ---------------------------------------------------------------------------------- |
| **Connected, not isolated**          | Copy and navigation preserve World context                                         |
| **Filter, not fork**                 | Same blocks, same header, same template                                            |
| **Observation before participation** | Registration Gateway remains after Evidence                                        |
| **Honest scope**                     | Sparse national activity shown honestly — not inflated                             |
| **Dignity without politics**         | Country pages reveal public civic records — they do not perform statehood advocacy |

Country Experience **strengthens local understanding** while preserving Humanity Union's **global perspective**.

Visitors should feel they are **still inside Humanity Union** — observing one country's slice of the same society window.

# 6. National Identity

Discovery explores architectural representation of national Identity **without changing the page template**.

All items below belong to the **Identity and Context Introduction layers** — not Evidence manipulation for persuasion.

## National flag

- **Architectural role:** optional Identity visual context — signals which country scope is active.
- **Discovery questions:** Is flag display architecturally required or optional? How is neutrality preserved for disputed or multi-representational contexts?
- **Constraint:** flag supports **where am I** — not **who is legitimate**.

## Country name

- **Architectural role:** primary Identity label in Hero and Geographic Navigator.
- **Discovery intent:** canonical country naming from public-safe geographic metadata — not UI invention per implementation.

## National symbols

- **Architectural role:** if used — minimal, Identity-layer only; never substitute for Evidence.
- **Constraint:** symbols must not imply official endorsement by Humanity Union or certification of national civic authority.

## Country-specific Context Introduction

- **Architectural role:** explains why **this country's** public civic activity matters in Humanity Union — one or two calm sentences below Hero title.
- **Tone discipline:** same reference tone as Epic 02 frozen Context Introduction standard — significance without pressure.
- **Discovery output:** content architecture document — not marketing copy in discovery.

## National civic statistics

- **Architectural role:** Statistics block Evidence — Participation Public Statistics Projection at country scope.
- **Constraint:** derived values labeled; honest zero or low activity; no ranking of nations.

## National initiatives

- **Architectural role:** Latest Initiatives Evidence — Initiative Public Projection list filtered to country.
- **Constraint:** same card responsibility as Global Experience — title, stage, public status, link to public detail.

## Trusted national media

- **Architectural role:** optional Exploration or secondary Evidence — Verified Media capability public associations at country scope.
- **Constraint:** not a substitute for Participation projections; optional block only if Block Library and future Media epic authorize.

**Template unchanged.**

National Identity adapts **Identity and Context layers** and **filtered Evidence datasets** — not page structure.

# 8. Relationship with Future Region Experience

Country Experience is the **parent scope** for Region Experience in the frozen geographic hierarchy.

```
World (Global Experience)

↓

Country (Country Experience)   ← this Epic

↓

Region (Region Experience)     ← future Epic 04 discovery
```

| Transition       | Architectural meaning                                             |
| ---------------- | ----------------------------------------------------------------- |
| World → Country  | Broader to national filter — Identity updates, Evidence refreshes |
| Country → Region | National to local filter — same template, finer dataset           |
| Region → Country | Ascent — reversible, same chrome                                  |
| Country → World  | Return to global public square                                    |

Regional pages **inherit the same architectural language** as Country and Global Experience.

Discovery must ensure Country Experience:

- exposes Region as navigator and map exploration entry — not hidden behind new navigation;
- does not pre-implement Region pages in Epic 03 scope;
- establishes scope parameter and URL patterns that Region Experience can reuse without architectural fork.

Future local levels — district, municipality, city, indigenous territory — remain supported by hierarchy extensibility in Epic 01 freeze.

Epic 03 discovery must not assume a fixed region model — only that **Country sits above Region** in scope chain.

# 9. Preliminary Risks

Discovery identifies the following **preliminary risks** — to be addressed in architecture review, not ignored at specification time.

## National isolation

**Risk:** Country pages feel like separate national portals disconnected from Humanity Union.

**Mitigation direction:** preserve World ascent, global chrome, connected civic space language, unified header.

## Duplicated architecture

**Risk:** engineering introduces CountryPageTemplate, country-specific blocks or parallel navigation.

**Mitigation direction:** enforce Filter Instead of Duplicate; reuse Global Experience composition with scope parameter.

## Different navigation logic

**Risk:** country scope changes header destinations, hides primary nav or adds country-administration entries.

**Mitigation direction:** frozen six destinations; Geographic Navigator handles scope — not header fork.

## Fragmented terminology

**Risk:** UI introduces country-specific synonyms for architectural blocks — breaking canonical naming registry.

**Mitigation direction:** architectural names unchanged; UI labels may read "National Statistics" — block remains Statistics family.

## Political presentation

**Risk:** national flags, symbols or copy imply political endorsement, ranking or conflict positioning.

**Mitigation direction:** Identity context only; Evidence from projections; Public Space never persuades — it reveals.

## Violation of Public Space principles

**Risk:** national urgency, registration pressure or fabricated activity to appear "active" at country scope.

**Mitigation direction:** observation precedes participation; honest empty states; calm Registration Gateway; derived labeling.

These risks are **discovery warnings** — not assumed failures.

Architecture review must close them before freeze.

# 10. Success Criteria

Country Experience discovery succeeds when downstream architecture can satisfy:

## Immediate familiarity

A visitor who knows Global Experience should **immediately recognize** Country Experience:

- the same block sequence and page rhythm;
- the same header and footer;
- the same calm, readable public tone;
- the same Context Before Evidence structure in every block.

No relearning required for **where things are**.

Only **what data means at national scope** is new.

## Recognizable architecture

Visitors recognize:

| Element                       | Expected recognition                                           |
| ----------------------------- | -------------------------------------------------------------- |
| **Architecture**              | Same blocks, same order, same template                         |
| **Interaction model**         | Map and navigator change scope; links explore; no forced paths |
| **Trust model**               | Derived labels, public projections, honest sparsity            |
| **Humanity Union philosophy** | One connected civic space — country is a lens, not a wall      |

## Natural national understanding

Visitors naturally understand:

**"I am observing civic activity associated with this country within Humanity Union."**

Success includes visitors who:

- ascend to World without disorientation;
- descend toward Region when available;
- explore initiatives without registering;
- evaluate trust through Evidence — not national rhetoric.

Thirty-second orientation at Country scope measures **national place recognition** — not mastery of all regional data.

# 11. Final Statement

**Country Experience is the national expression of Humanity Union Public Space.**

It preserves **one architectural language** while presenting **one country's observable civic reality**.

Country Experience is Geographic Experience at Country scope — the next step in the frozen hierarchy after Global Experience at World scope.

It extends what Epic 02 built.

It does not replace it.

It does not fork it.

Downstream artifacts — Country Experience vision, content architecture, interaction architecture, page specification, architecture review and implementation plan — must conform to this discovery and to frozen Epic 01 and Epic 02 documents.

**No implementation is defined by this document.**

Implementation waits for architecture.

Architecture waits for honest answers to the questions above.
