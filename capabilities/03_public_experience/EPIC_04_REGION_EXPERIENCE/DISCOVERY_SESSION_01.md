# 1. Discovery Purpose

Initiate the **architectural discovery** of **Region Experience** within Humanity Union Public Space.

Region Experience presents the **civic activity of one administrative region** while preserving Humanity Union's unified architectural language.

It is **not** an independent platform.

It is **not** a separate product, site or navigation universe.

It is a **filtered public representation of one region within one country** within the unified Humanity Union architecture — architecturally named **Geographic Experience at Region scope**.

This discovery session defines the **architectural role** Region Experience must fulfill before specification, content architecture, interaction architecture and implementation planning begin.

Region Experience **extends Country Experience** without introducing a new architectural language.

Epic 03 delivered Country Experience at Country scope and frozen the geographic hierarchy through Region toward Community.

Epic 04 discovery asks how the **same frozen architecture** expresses observable civic reality when the visitor's scope is one region within one country.

**Region Experience narrows geographic scope from Country to Region without changing Humanity Union architecture.**

This document records discovery intent only.

It does not define implementation.

It does not authorize code, routes, components, visual design or Capability 02 backend changes.

# 3. Architectural Questions

Discovery must explore and later answer the following questions — without prescribing implementation.

## What distinguishes Region Experience from Country Experience?

- **Scope parameter** — visitor observes civic activity associated with one region within one country, not the entire national aggregate.
- **Identity framing** — Hero and page context name the region **within its country** as a participating place within Humanity Union — not as an isolated local portal.
- **Evidence density** — statistics, pipeline distribution, initiatives and map markers reflect regional public activity — may include honest sparse or empty states for regions with limited public records.
- **Exploration entry** — map and navigator support ascent to Country and World scope; exploration block prepares **Community Experience** descent — not further administrative subdivision in Version 1 discovery.
- **Parent context** — region is always locatable within a named country — Country context preserved in navigator and Identity layer.

Region Experience differs in **dataset and contextual meaning** — not in page anatomy or trust model.

## Which information is inherited?

Region Experience **inherits** from Country Experience and upstream freezes:

| Inherited element       | Region behavior                                                                                  |
| ----------------------- | ------------------------------------------------------------------------------------------------ |
| Block sequence pattern  | Same order and responsibility model as Country Experience — region-labeled                       |
| Block responsibility    | One Experience Block — One Responsibility                                                        |
| Header destinations     | Same six primary destinations                                                                    |
| Geographic Navigator    | Region active within Country; Country and World ascent                                           |
| Page template flow      | Identity → Context Introduction → Evidence → Exploration → Participation → Supporting Navigation |
| Interaction philosophy  | Every interaction increases understanding                                                        |
| Learning Path           | Observe → Understand → Explore → Evaluate → Participate                                          |
| Projection boundary     | Public projections only — region-filtered                                                        |
| Registration ethics     | Calm invitation after regional observation                                                       |
| Footer                  | Supporting navigation — not second conversion stack                                              |
| Context Before Evidence | Heading → Context Introduction → Evidence → Visitor Conclusion                                   |

```
Country Experience + region scope parameter → Region Experience
```

Epic 03 Country Experience is the **direct inheritance reference** — Epic 02 Global Experience remains the original template reference.

## Which information becomes region-specific?

Discovery candidates — subject to later content architecture:

- region name and scope label in Identity layer — **within country**;
- region Context Introduction — significance of regional civic activity within national and global Humanity Union context;
- regional public statistics from filtered projections;
- regional participation pipeline distribution;
- latest initiatives associated with the region;
- map Evidence showing civic activity **within the region**;
- optional regional media where Verified Media capability supplies public-safe region associations;
- Community Exploration entry — scope transition preparing Community Experience where public community associations exist.

Region-specific copy must remain **explanatory** — not promotional, administrative boosterism or certifying.

## How does Region Experience prepare visitors for Community Experience?

Discovery must resolve:

- Region is the **geographic bridge** between national observation and participant-named local civic context per `COMMUNITY_CONTEXT_DECISION.md`;
- Region page may **route toward** Community discovery where public associations exist — but **Find Your Community** remains Community Experience primary entry;
- visitor must understand Region is **administrative or regional geography** — not yet **participant-named Community**;
- Community blocks, community search and community-specific statistics **must not** appear on Region page as primary composition;
- scope transition block at Region scope prepares Community descent — analogous to Regional Exploration at Country scope preparing Region descent.

Region **prepares** Community.

Region **does not implement** Community Experience.

## Which data continues to originate from Capability 02 public projections?

Per `CAPABILITY_02_PROJECTION_INTEGRATION.md`, Region Experience consumes the **same architectural projection contracts** filtered by region scope:

| Block                  | Projection source                                          |
| ---------------------- | ---------------------------------------------------------- |
| Interactive Map        | Public Geographic Participation Projection — region filter |
| Statistics             | Participation Public Statistics Projection — region filter |
| Participation Pipeline | Participation Pipeline Public Projection — region filter   |
| Latest Initiatives     | Initiative Public Projection — region filter               |

Capability 03 **orchestrates and filters**.

Capability 03 **does not** access operational aggregates.

Capability 02 **owns** aggregation rules and public-safe geographic metadata in projection builders.

## How is regional identity represented without fragmenting Humanity Union?

Discovery must resolve:

- region named clearly **within country context** without implying Humanity Union is a federation of separate local portals;
- representative regional image — if present — as **Identity orientation**, not tourism marketing;
- navigation always preserves return path to Country, World and global chrome;
- language that presents the region as **one participating place within one country within one connected civic space**.

Regional Identity strengthens **local comprehension**.

It must not weaken **national or global coherence**.

# 4. Visitor Perspective

## Why would someone visit a Region Experience page?

Typical intentions:

- understand civic activity **in my region** or **in a region I am researching** within a known country;
- move from national overview to **regional context** before community or initiative detail;
- evaluate whether structured participation exists **where I live regionally**;
- follow a map, Regional Exploration block or navigator transition from Country to Region deliberately;
- share or bookmark a **region-scoped public URL** as a stable civic reference;
- continue progressive local exploration toward **participant-named communities** after understanding regional civic activity.

Region Experience serves **regional orientation within national and global architecture** — not regional exit from Humanity Union.

## What questions should it answer?

Region Experience should naturally answer:

| Question                                               | Architectural responsibility                                                                  |
| ------------------------------------------------------ | --------------------------------------------------------------------------------------------- |
| **Where am I?**                                        | Region Identity — Hero, scope label, Geographic Navigator — region within country             |
| **Why does this region's civic activity matter here?** | Context Introduction — regional significance within country and Humanity Union                |
| **What is happening in this region?**                  | Evidence — statistics, pipeline, initiatives, map                                             |
| **Where can I explore next?**                          | Exploration — communities, related initiatives, header destinations, Country and World return |
| **How may I participate?**                             | Registration Gateway — optional, after observation                                            |

These mirror the Five Visitor Questions from `PUBLIC_PAGE_TEMPLATE_STANDARD.md` — unchanged at Region scope.

Primary observation question at Region scope:

**What is happening in this region?**

Region Experience must **not** answer **what is happening around this community** — that belongs to Community Experience.

## What should visitors understand before moving into Community Experience?

Before Community descent, visitors should understand:

| Understanding                                     | Architectural meaning                                                                                  |
| ------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **Scope difference**                              | Region is geographic/administrative filter — Community is participant-named civic context              |
| **Observation completed at regional level**       | Regional statistics, pipeline and initiatives have been presented honestly                             |
| **Community is discovered through participation** | Communities are not an administrator-only catalog — Find Your Community is the primary Community entry |
| **Same architecture continues**                   | Community Experience uses the same block language — finer civic context, not a new product             |
| **Ascent remains available**                      | Region → Country → World — reversible without penalty                                                  |
| **Participation remains optional**                | Registration Gateway invitation follows observation — not a gate before Community exploration          |

Region prepares **progressively local civic understanding**.

Community Experience **completes** local observation before optional Workspace participation.

# 5. Geographic Philosophy

Discovery confirms the following geographic philosophy for Region Experience.

**A region is part of one country.**

**It strengthens local understanding while preserving Humanity Union's global perspective.**

Humanity Union presents regional civic activity as **locatable within a national place within a global public square** — not as sealed regional silos.

| Principle                                  | Region Experience expression                                                               |
| ------------------------------------------ | ------------------------------------------------------------------------------------------ |
| **Connected, not isolated**                | Copy and navigation preserve Country and World context                                     |
| **Part of one country**                    | Region Identity always locates region within named country                                 |
| **Filter, not fork**                       | Same blocks, same header, same template                                                    |
| **Observation before participation**       | Registration Gateway remains after Evidence                                                |
| **Honest scope**                           | Sparse regional activity shown honestly — not inflated                                     |
| **Dignity without administration worship** | Region pages reveal public civic records — they do not perform regional authority advocacy |
| **Bridge to Community**                    | Region narrows geography before participant-named Community context                        |

Region Experience **strengthens local understanding** while preserving Humanity Union's **global perspective** and **national context**.

Visitors should feel they are **still inside Humanity Union** — observing one region's slice of the same society window.

Region is **narrower than Country** — not **more authoritative**.

# 6. Regional Identity

Discovery explores architectural representation of regional Identity **without changing the page template**.

All items below belong to the **Identity and Context Introduction layers** — not Evidence manipulation for persuasion.

## Region name

- **Architectural role:** primary Identity label in Hero and Geographic Navigator — **qualified by country context**.
- **Discovery intent:** canonical region naming from public-safe geographic metadata — not UI invention per implementation.
- **Discovery questions:** How is region name displayed relative to country name — subtitle, breadcrumb, navigator label? How is ambiguity handled when region names repeat across countries?

## Representative image

- **Architectural role:** optional Identity visual context — landscape, cityscape or regional landmark supporting **where am I** orientation.
- **Constraint:** atmospheric orientation — not tourism marketing or regional promotional campaign.
- **Discovery questions:** Is representative image required or optional? How is neutrality preserved when regional imagery carries cultural or political connotation?

## Context Introduction

- **Architectural role:** explains why **this region's** public civic activity matters in Humanity Union — within **this country** — one or two calm sentences below Hero title.
- **Tone discipline:** same reference tone as Epic 02 and Epic 03 Context Introduction standards — significance without pressure.
- **Constraint:** explains significance — never persuades; never replaces Evidence.
- **Discovery output:** content architecture document — not marketing copy in discovery.

## Regional statistics

- **Architectural role:** Statistics block Evidence — Participation Public Statistics Projection at region scope.
- **Constraint:** derived values labeled; honest zero or low activity; no ranking of regions against each other as civic virtue competition.

## Regional initiatives

- **Architectural role:** Latest Initiatives Evidence — Initiative Public Projection list filtered to region.
- **Constraint:** same card responsibility as Country and Global Experience — title, stage, public status, link to public detail.

## Regional media (Version 1 discussion)

- **Architectural role:** optional Exploration or supporting Evidence — Verified Media capability public associations at region scope — analogous to Trusted National Media at Country scope.
- **Discovery questions for Version 1:**
  - Is **Trusted Regional Media** included in Region Experience composition — mirroring Country optional block pattern?
  - Or deferred entirely from Version 1 Region Experience as Community Experience deferred Trusted Media?
  - If included — same optional omission semantics when no public-safe regional media exists?
- **Constraint:** not a substitute for Participation projections; governed source — not platform truth claim; omission is honest when no data exists.

**Template unchanged.**

Regional Identity adapts **Identity and Context layers** and **filtered Evidence datasets** — not page structure.

Discovery must later resolve Community Exploration block naming and responsibility — analogous to Regional Exploration at Country scope — in content architecture; not in this discovery freeze.

# 9. Preliminary Risks

Discovery identifies the following **preliminary risks** — to be addressed in architecture review, not ignored at specification time.

## Architecture duplication

**Risk:** engineering introduces RegionPageTemplate, region-specific blocks or parallel navigation instead of scope filter on Country Experience pattern.

**Mitigation direction:** enforce Filter Instead of Duplicate; reuse Country Experience composition with region scope parameter; Epic 03 freeze is binding reference.

## Navigation inconsistency

**Risk:** region scope changes header destinations, hides Country ascent or elevates Community to equal chrome sibling incorrectly at Region scope.

**Mitigation direction:** frozen six destinations; Geographic Navigator handles scope — Country and World ascent preserved; Community not primary navigator peer at Region chrome in Version 1 alignment.

## Administrative dependence

**Risk:** Region Experience over-identifies with government administrative boundaries — excluding civic activity that spans regions or mislabeling participant Communities as administrative regions.

**Mitigation direction:** Region presents projection-backed Evidence honestly; Community remains participant-named; architecture does not equate administrative region with Community; culturally sensitive boundary labeling requires separate governance review.

## Overlapping responsibilities

**Risk:** Regional Interactive Map, Regional Statistics and exploration block duplicate the same region list or initiative catalog under alternate headings.

**Mitigation direction:** One Experience Block — One Responsibility; Filter Instead of Duplicate; cross-reference links permitted — duplicated primary Evidence forbidden.

## Community confusion

**Risk:** visitors interpret Region page as Community page — especially when region names match city or community names (e.g. "Nelson" as region vs "Nelson" as Community).

**Mitigation direction:** scope labels visible; Context Introduction explains Region vs Community distinction; Find Your Community not on Region page; Community Exploration block explains participant-named context ahead.

## Regional isolation

**Risk:** Region pages feel like separate local portals disconnected from Country and Humanity Union.

**Mitigation direction:** preserve Country and World ascent, global chrome, connected civic space language, country name in Identity layer.

## Sparse data trust erosion

**Risk:** low-activity regions appear abandoned or untrustworthy if empty states poorly written — or conversely fabricated activity to appear vibrant.

**Mitigation direction:** honest sparse states; derived labeling; Explainable Honesty; no national or regional boosterism in copy.

## Political or administrative advocacy

**Risk:** regional symbols, copy or map boundaries imply political endorsement, territorial dispute positioning or administrative hierarchy worship.

**Mitigation direction:** Identity context only; Evidence from projections; Public Space never persuades — it reveals; boundary sensitivity deferred to governance review.

These risks are **discovery warnings** — not assumed failures.

Architecture review must close them before freeze.

# 10. Success Criteria

Region Experience discovery succeeds when downstream architecture can satisfy:

## Immediate familiarity

A visitor who knows Global Experience and Country Experience should **immediately recognize** Region Experience:

- the same block sequence pattern and page rhythm;
- the same header and footer;
- the same calm, readable public tone;
- the same Context Before Evidence structure in every block.

No relearning required for **where things are**.

Only **what data means at regional scope** is new.

## Recognizable architecture

Visitors recognize:

| Element                       | Expected recognition                                                      |
| ----------------------------- | ------------------------------------------------------------------------- |
| **Architecture**              | Same blocks, same order pattern, same template                            |
| **Interaction model**         | Map and navigator change scope; links explore; no forced paths            |
| **Trust model**               | Derived labels, public projections, honest sparsity                       |
| **Humanity Union philosophy** | One connected civic space — region is a lens within a country, not a wall |
| **Country context**           | Region locatable within named country at all times                        |

## Natural regional understanding

Visitors naturally understand:

**"I am observing civic activity associated with this region within this country within Humanity Union."**

Success includes visitors who:

- ascend to Country and World without disorientation;
- understand Region is not yet Community;
- route toward Community Experience when ready — voluntarily;
- explore initiatives without registering;
- evaluate trust through Evidence — not regional rhetoric.

Thirty-second orientation at Region scope measures **regional place recognition within country** — not mastery of all community data.

## Community preparation success

Visitors who continue to Community Experience should carry:

- understanding of regional civic activity observed;
- clear scope distinction between Region and Community;
- expectation that Community discovery is participant-driven — not administrative directory browsing.

It does not fork it.
