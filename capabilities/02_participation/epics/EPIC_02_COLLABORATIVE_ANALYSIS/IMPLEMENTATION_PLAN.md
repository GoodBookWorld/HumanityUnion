- Domain Ownership
- Progressive Bootstrap
- Human Leadership

- Explainability by Default

- Read before Write
- Manual before Automation
- Foundation before Capability

# Implementation Constraints

The implementation shall not:

- redefine domain language;
- modify accepted domain decisions;
- bypass Progress Policy;
- edit immutable Contributions;
- violate Human Leadership;
- move intelligence into business logic.

---

# Integration Rules

Collective Intelligence services remain optional during bootstrap.

Business functionality must remain operational without advanced intelligence features.

Advanced services may enhance implementation later.

---

# Success Criteria

Epic implementation is successful when:

- architecture is preserved;
- Contributions are immutable;
- analysis remains additive;
- Progress Policy governs readiness;
- Poll receives only prepared Initiatives;
- platform foundations remain unchanged.
